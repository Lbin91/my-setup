## 🤔 이게 뭔데?

### 한 문장 요약
> **Claude를 "한 번에 하나씩 일하는 AI"에서 "여러 일을 동시에 처리하는 슈퍼 AI"로 바꿔주는 플러그인**

### 비유로 이해하기

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   [일반 Claude]              [Oh-My-Claude 적용 후]            │
│                                                                 │
│   👤 → 📝 → 📝 → 📝 → ✅      👤 → 📝📝📝📝 → ✅              │
│                                    📝📝📝📝                    │
│   혼자서 순서대로 처리         여러 팀원이 동시에 처리!         │
│   (시간 오래 걸림)             (훨씬 빠름!)                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**예를 들면:**
- 🍔 **일반 Claude** = 혼자 요리하는 셰프 (재료 손질 → 굽기 → 플레이팅 순서대로)
- 🍔 **Oh-My-Claude** = 10명의 전문 셰프 팀 (동시에 각자 맡은 일 처리!)

## 🎯 핵심 개념 3가지

### 1️⃣ 병렬 실행 (Parallel Execution)

```
┌────────────────────────────────────────────────────┐
│                                                    │
│  ❌ 순차 실행 (기존)                               │
│  ┌──────┐    ┌──────┐    ┌──────┐                 │
│  │작업 1│ → │작업 2│ → │작업 3│  = 30분          │
│  └──────┘    └──────┘    └──────┘                 │
│                                                    │
│  ✅ 병렬 실행 (Oh-My-Claude)                       │
│  ┌──────┐                                         │
│  │작업 1│ ─┐                                      │
│  └──────┘  │                                      │
│  ┌──────┐  ├→ 동시 완료!  = 10분                  │
│  │작업 2│ ─┤                                      │
│  └──────┘  │                                      │
│  ┌──────┐ ─┘                                      │
│  │작업 3│                                         │
│  └──────┘                                         │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 2️⃣ 컨텍스트 보호 (Context Protection)

**컨텍스트란?** Claude가 대화 중 기억할 수 있는 정보의 양 (한정됨!)

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  🧠 Claude의 뇌 = 컨텍스트 창 (Context Window)              │
│                                                              │
│  ┌────────────────────────────────────────────────────┐     │
│  │ ████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │     │
│  │ [사용중 40%]          [남은 공간 60%]               │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  ⚠️ 큰 파일을 직접 읽으면?                                  │
│  ┌────────────────────────────────────────────────────┐     │
│  │ ██████████████████████████████████████████████████ │     │
│  │ [꽉 참! 더 이상 생각할 공간 없음!]                   │     │
│  └────────────────────────────────────────────────────┘     │
│                                                              │
│  ✅ Oh-My-Claude는 큰 파일을 "대리인"에게 맡김!              │
│     → 메인 Claude의 뇌는 "생각"만 하면 됨!                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**규칙:**
| 파일 크기 | 어떻게 처리? |
|-----------|--------------|
| 100줄 미만 | 직접 읽기 OK |
| 100줄 이상 | 🚫 직접 읽기 금지! → `librarian`에게 위임 |
| 여러 파일 | 🚫 무조건 위임! |

### 3️⃣ 완료할 때까지 멈추지 않음

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  일반 Claude                  Oh-My-Claude          │
│                                                      │
│  "여기까지 했어요,            "모든 작업 완료 후    │
│   다음 뭐 할까요?"             검증까지 끝나면       │
│        ↓                       멈춥니다"             │
│  [매번 물어봄]                     ↓                │
│                               [자동으로 끝까지]      │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## 🪄 마법의 키워드 (이것만 기억해!)

프롬프트에 이 키워드를 넣으면 특수 모드가 활성화됩니다!

### 🔥 ultrawork (줄여서: ulw)

> **"최대 병렬 실행 모드"** - 가장 많이 쓰는 키워드!

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  📝 사용법:                                                 │
│  "ultrawork로 로그인 시스템 만들어줘"                       │
│  또는                                                       │
│  "ulw 이 프로젝트에 테스트 코드 추가해줘"                   │
│                                                             │
│  ✨ 효과:                                                   │
│  • 여러 에이전트가 동시에 작업 시작                         │
│  • Todo 리스트로 진행 상황 자동 추적                        │
│  • 모든 작업 + 검증 완료될 때까지 안 멈춤                   │
│  • 질문 없이 스스로 결정하고 진행                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**일반 모드 vs Ultrawork 모드 비교:**

| 구분 | 일반 모드 | Ultrawork 모드 |
|------|----------|----------------|
| 실행 방식 | 순차적 (하나씩) | 병렬 (동시에!) |
| 파일 읽기 | 직접 읽음 | 대리인에게 위임 |
| 진행 추적 | 선택적 | 필수 (Todo 사용) |
| 멈추는 시점 | 각 단계마다 | 모든 작업 완료 후 |
| 결과물 | 부분 완성 OK | 완전한 완성만 OK |
| 권한 요청 | 자주 물어봄 | 안전한 건 자동 승인 |

## 👥 10명의 AI 팀원들

Oh-My-Claude는 10명의 전문 에이전트가 팀으로 일합니다!

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│                        🎭 AI 에이전트 팀                           │
│                                                                    │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │                                                              │ │
│  │   🔍 scout        📚 librarian     👁️ looker                │ │
│  │   "파일 찾기"      "큰 파일 읽기"    "이미지/PDF 분석"       │ │
│  │                                                              │ │
│  │   🏗️ architect    🧐 critic        🐛 debugger              │ │
│  │   "계획 설계"      "계획 검토"       "버그 해결"              │ │
│  │                                                              │ │
│  │   👷 worker       📝 scribe        ✅ validator             │ │
│  │   "실제 구현"      "문서 작성"       "테스트/검증"           │ │
│  │                                                              │ │
│  │                    🎯 orchestrator                           │ │
│  │                    "전체 조율/지휘"                          │ │
│  │                                                              │ │
│  └──────────────────────────────────────────────────────────────┘ │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 각 에이전트 상세 설명

| 에이전트 | 역할 | 언제 사용? | 비유 |
|----------|------|-----------|------|
| 🔍 **scout** | 파일 찾기, Git 정보 조사 | 코드베이스 탐색할 때 | 정찰병 |
| 📚 **librarian** | 큰 파일 읽기, 요약 | 100줄 넘는 파일 읽을 때 | 사서 |
| 👁️ **looker** | 이미지, PDF, 다이어그램 분석 | 시각적 자료 볼 때 | 감정사 |
| 🏗️ **architect** | 복잡한 작업 분해, 계획 | 큰 프로젝트 설계할 때 | 건축가 |
| 🧐 **critic** | 계획 검토, 문제점 발견 | 실행 전 검증할 때 | 비평가 |
| 🐛 **debugger** | 에러 해결 (2회 이상 실패 시) | 버그가 안 잡힐 때 | 탐정 |
| 👷 **worker** | 실제 코드 구현 | 코딩할 때 | 개발자 |
| 📝 **scribe** | 문서, README 작성 | 문서화할 때 | 작가 |
| ✅ **validator** | 테스트, 린터 실행 | 품질 검사할 때 | 검수관 |
| 🎯 **orchestrator** | 여러 에이전트 조율 | 복잡한 워크플로우 | 지휘자 |

### 에이전트 협업 흐름 예시

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  사용자: "ulw 로그인 기능 만들어줘"                             │
│                                                                 │
│  ┌─────┐   ┌─────────┐   ┌──────────┐   ┌───────┐              │
│  │scout│ → │librarian│ → │architect │ → │critic │              │
│  │파일 │   │코드     │   │계획      │   │검토   │              │
│  │찾기 │   │분석     │   │수립      │   │       │              │
│  └─────┘   └─────────┘   └──────────┘   └───────┘              │
│                                              ↓                  │
│  ┌─────────┐   ┌──────┐   ┌──────────────────────┐             │
│  │validator│ ← │worker│ ← │ 계획 승인! 구현 시작 │             │
│  │테스트   │   │구현  │   └──────────────────────┘             │
│  │검증     │   │      │                                        │
│  └─────────┘   └──────┘                                        │
│       ↓                                                         │
│  ✅ 완료!                                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 🔧 자동으로 작동하는 Hooks

Hooks는 **백그라운드에서 자동으로 작동하는 규칙들**입니다.
사용자가 따로 설정할 필요 없이, 플러그인 설치만 하면 자동 적용!

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  🎣 Hooks = "자동 감시 & 보호 시스템"                            │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │  당신이 입력 →  [Hooks가 자동 검사]  → Claude가 실행      │ │
│  │                       ↓                                    │ │
│  │              필요시 자동 개입/보호                         │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 주요 Hooks 설명

| Hook 이름 | 하는 일 | 비유 |
|-----------|--------|------|
| 🛡️ **context_guardian** | 세션 시작 시 보호 규칙 주입 | 경비원 |
| 🔍 **ultrawork_detector** | ulw/ulr/uld 키워드 감지 | 감시 카메라 |
| 🚫 **context_protector** | 큰 파일 직접 로드 차단 | 보안 게이트 |
| ✅ **safe_permissions** | 안전한 명령어 자동 승인 | 자동문 |
| 📋 **todo_enforcer** | 미완료 작업 있으면 종료 방지 | 체크리스트 |
| 📊 **context_monitor** | 컨텍스트 사용량 경고 | 연료 게이지 |
| 🔒 **danger_blocker** | 위험한 명령어 차단 | 비상 정지 버튼 |
| 🧪 **subagent_quality_validator** | 서브에이전트 결과물 검증 | 품질 검사관 |
| 💾 **precompact_context** | 컨텍스트 압축 전 상태 저장 | 자동 저장 |
| 🔔 **notification_alert** | 알림 전송 (선택적) | 알림 벨 |

### Hook 동작 시각화

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  💬 "ulw 프로젝트 분석해줘" 입력                                 │
│           ↓                                                      │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ 🔍 ultrawork_detector                                      │ │
│  │    "ulw 키워드 발견! Ultrawork 모드 활성화!"               │ │
│  └────────────────────────────────────────────────────────────┘ │
│           ↓                                                      │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ 🛡️ context_guardian                                        │ │
│  │    "보호 규칙 주입 완료!"                                  │ │
│  └────────────────────────────────────────────────────────────┘ │
│           ↓                                                      │
│  🤖 Claude 작업 시작...                                          │
│           ↓                                                      │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ 🚫 context_protector                                       │ │
│  │    "500줄 파일 직접 읽기 시도? → 차단! librarian에게 위임" │ │
│  └────────────────────────────────────────────────────────────┘ │
│           ↓                                                      │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ 📋 todo_enforcer                                           │ │
│  │    "아직 3개 작업 남음! 종료 불가!"                        │ │
│  └────────────────────────────────────────────────────────────┘ │
│           ↓                                                      │
│  ✅ 모든 작업 완료 + 검증 통과 → 종료 허용                       │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## ⚙️ 설정 옵션

settings.json에서 환경변수로 설정 가능:

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  📁 settings.json 설정 예시                                      │
│                                                                  │
│  {                                                               │
│    "env": {                                                      │
│      "OMC_LARGE_FILE_THRESHOLD": "100",   // 큰 파일 기준 (줄)  │
│      "OMC_CONTEXT_WARN_PCT": "70",        // 경고 시점 (%)      │
│      "OMC_CONTEXT_CRITICAL_PCT": "85",    // 위험 시점 (%)      │
│      "OMC_DANGER_BLOCK": "1",             // 위험명령 차단      │
│      "OMC_NOTIFICATIONS": "1"             // 알림 활성화        │
│    }                                                             │
│  }                                                               │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 설정값 상세 설명

| 환경변수 | 기본값 | 설명 |
|----------|--------|------|
| `OMC_LARGE_FILE_THRESHOLD` | 100 | 이 줄수 이상이면 librarian에게 위임 |
| `OMC_CONTEXT_WARN_PCT` | 70 | 컨텍스트 70% 사용 시 경고 |
| `OMC_CONTEXT_CRITICAL_PCT` | 85 | 컨텍스트 85% 사용 시 위험 경고 |
| `OMC_DANGER_BLOCK` | 1 | 위험한 명령어 자동 차단 (rm -rf 등) |
| `OMC_NOTIFICATIONS` | 0 | 1로 설정 시 알림 활성화 |
